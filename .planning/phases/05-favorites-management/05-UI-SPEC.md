## Color

Native SwiftUI system colors (follows system dark mode, already established):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `systemBackground` | Main background, page surfaces |
| Secondary (30%) | `ultraThinMaterial`, `secondarySystemBackground` | Cards, overlays, list cell backgrounds |
| Accent (10%) | `accentColor` (system) | Focused rows, text field focus indicator, interactive elements |
| Destructive | `red` | Delete action buttons |

Accent reserved for:
- Focused list row highlighting
- Add button when input is valid
- Focus effect on text field

## Layout Patterns (from 05-CONTEXT.md decisions)

Locked decisions from upstream context:

1. **Favorites List**: Plain `List` with full-width rows (standard tvOS pattern)
2. **Cell Content**: Room title, author nickname, colored live indicator, last checked time
3. **Selection Behavior**: Immediate navigation to WatchLive tab on selection
4. **Sort Order**: By `lastViewed` descending (most recently used at top)
5. **Delete Interaction**: Through edit mode with delete buttons (standard tvOS pattern)
6. **Add Room**: Text field with dictation support (already enabled from Phase 3)
7. **Post-Add Navigation**: Switch to Favorites tab after successful add
8. **Single Selection**: Only one room active at a time (matches project scope)

---

## Interaction Contracts (tvOS Focus)

Inherited from Phase 3 foundation (already required by FAV-01 to FAV-06):

- All interactive items minimum 88x88pt focus targets
- `focusable()` modifier with system `focusEffect()`
- Supports standard Siri Remote gestures: swipe to navigate, click to select, Menu to exit edit mode
- Follows existing tvOS HIG already established in Phase 3

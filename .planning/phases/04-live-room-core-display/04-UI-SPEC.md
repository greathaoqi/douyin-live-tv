# Phase 4 — UI Design Contract

> Visual and interaction contract for the live room core display phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native SwiftUI) |
| Preset | not applicable (tvOS native) |
| Component library | SwiftUI native — manual design system implementation (no third-party registry) |
| Icon library | SF Symbols (system) |
| Font | System San Francisco (tvOS default) |

**Explicit declaration:** This is a native SwiftUI project with manual design system implementation. No shadcn or other third-party component registry is used. All components are built from scratch using native SwiftUI APIs.

---

## Spacing Scale

Declared values follow 8-point grid (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, stats item spacing |
| lg | 24px | Vertical spacing between stats, section padding |
| xl | 32px | Layout gaps, overlay container padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: All focusable buttons must meet **minimum 88pt size** per tvOS HIG (already established in Phase 3)

---

## Typography

Following CONTEXT.md decisions for couch-distance readability:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 28px | regular (400) | 1.5 |
| Label (stats labels) / Heading (room title) / Button | 36px | regular (400) for labels, semibold (600) for heading/button | 1.2 |
| Value (stats values) | 48px | semibold (600) | 1.1 |

---

## Color

Uses native system colors for automatic dark mode support:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | .systemBackground / .black | Full-screen video background |
| Secondary (30%) | .secondarySystemBackground | Overlay background (with blur) |
| Accent (10%) | .accentColor (system) | Toggle button, interactive elements |
| Live indicator | .green | Live status badge |
| Offline indicator | .red | Offline status badge |
| Error | .systemRed | Error messages |

Accent reserved for:
- Overlay toggle button
- Focus highlights
- Status indicators

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Toggle Overlay |
| Empty state heading | No Live Room Selected |
| Empty state body | Select a live room from favorites or add a new one to start monitoring. |
| Error state | Could not load live room data. Check the room ID and try again. |
| Destructive confirmation | None in this phase |

---

## Interaction Contract (tvOS-specific)

From requirements and CONTEXT.md:

| Interaction | Behavior |
|-------------|----------|
| Focus navigation | All interactive elements meet 88pt minimum focus size |
| Play/Pause button | Toggles video playback (per tvOS standard established Phase 3) |
| Menu button | Navigates back per system standard |
| Overlay toggle | Toggle button switches between overlay mode and full-screen video |
| PiP activation | Picture in Picture starts automatically when app enters background |
| App lifecycle | Pause video on background, resume on foreground (uses AppLifecycleService from Phase 3) |

---

## Layout Contract (from CONTEXT.md decisions)

| Area | Specification |
|------|---------------|
| Video | Full-screen, .resizeAspectFill (fills screen, crops minor edges for 9:16 vertical streams) |
| Stats overlay | Top-leading (top-left) corner, semi-transparent blurred background, respects safe area |
| Overlay toggle button | Top-trailing (top-right) corner, opposite overlay, 88pt minimum size |
| Live/offline indicator | Badge next to room title at top of overlay |
| Statistics order | Viewer count → Likes → Total gifts |

# Phase 3 — UI Design Contract

## Spacing Scale

Declared values (multiples of 4 per tvOS convention):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon gaps, inline padding |
| sm | 8pt | Compact element spacing |
| md | 16pt | Default element spacing, inner padding |
| lg | 24pt | Section padding |
| xl | 32pt | Layout gaps between major blocks |
| 2xl | 48pt | Major section breaks |
| 3xl | 64pt | Page-level top/bottom spacing |

**Exceptions:** All focusable items must have minimum 88pt size per tvOS HIG requirement (UX-07). This overrides the token scale for interaction targets.

## Color

Uses system dynamic colors that automatically adapt to dark mode (per UX-08 decision to respect system setting).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | systemBackground | Main background, root surfaces |
| Secondary (30%) | secondarySystemBackground | Cards, tab bar, cell backgrounds |
| Accent (10%) | accentColor (system) | Tab selection, buttons, progress indicators, focus borders |
| Destructive | systemRed | Error messages, destructive actions |

**Accent reserved for:**
- Selected tab indicator
- Prominent buttons (CTAs)
- Progress activity indicators
- Focused item borders
- QR code container border (already used in LoginView)

## tvOS Specific Interaction Contract

These requirements are from UX-01 through UX-09 and locked in CONTEXT.md:

| Requirement | Contract |
|-------------|----------|
| Focus-based navigation | Use SwiftUI native `focusable()` modifier, let system manage focus propagation |
| Minimum focus target | 88x88pt minimum for all interactive elements (enforce via `frame(minWidth:minHeight:)`)|
| Parallax effect | Use native `.focusEffect()` modifier for system parallax on focus |
| Siri Remote gestures | Leave as system default: swipe moves focus, click selects, Menu navigates back |
| Play/Pause | System default will be inherited, no custom handling needed in this phase |
| Overscan insets | Apply `.safeAreaPadding()` to all root view content |
| Dictation | Enable on all text fields (Add Room screen, will be implemented in Phase 5) |
| Dark mode | No custom colors, use system dynamic colors — automatically respects system setting |
| App lifecycle | Observe `didEnterBackground` / `willEnterForeground` notifications in a helper |
| Focus restoration | Automatically restore focus to previously focused item when returning to a view |
